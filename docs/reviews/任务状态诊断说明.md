# 任务状态诊断说明 (Task Diagnosis Guide)

本说明用于指导 Lucia 和运维人员通过 `/cms/tasks` 列表中的“诊断矩阵”以及任务详情页的状态，快速判断任务是否处于健康生命周期。

## 1. 状态位定义

| 状态位 | 来源 | 含义 |
| :--- | :--- | :--- |
| **task.status** | DB (tasks) | 任务流转的核心状态（processing, completed, failed, review-pending） |
| **material.status** | DB (materials) | 资产回填状态 |
| **mineruStatus** | DB (materials) | MinerU 解析引擎的执行状态（pending, processing, completed, failed） |
| **aiStatus** | DB (materials) | AI 元数据识别的执行状态（pending, running, completed, failed） |

---

## 2. 合法状态组合 (Happy Path)

### 2.1 PDF 解析链路 (正常流转)
*   **上传后 (解析中)**: `processing / processing / processing / pending`
*   **解析完成 (待 AI)**: `processing / processing / completed / pending`
*   **AI 识别中**: `ai-running / processing / completed / running`
*   **AI 完成 (需人工核验)**: `review-pending / processing / completed / completed` (analyzed)
*   **最终归档 (已发布)**: `completed / completed / completed / completed` (analyzed)

### 2.2 Markdown 直通链路 (正常流转)
*   **上传后 (跳过 MinerU)**: `processing / processing / completed / pending`
*   **最终归档**: `completed / completed / completed / completed` (analyzed)

---

## 3. 异常诊断口径 (Audit required)

当诊断矩阵显示 **"Needs Audit"** 时，通常涉及以下情形：

| 表现 | 原因 | 处理建议 |
| :--- | :--- | :--- |
| **Stale Processing** | `task.status=processing` 且长时间（如 >10min）无变化 | 检查 Worker 容器是否在线。若是单机异常，可尝试手动重置状态（详见一致性清理手册）。 |
| **AI Running Timeout** | `aiStatus=running` 超过 2 分钟 | 系统会自动在 2 分钟后尝试重置为 pending。若仍卡住，需检查 Ollama 服务健康度。 |
| **Orphan Task** | 任务记录存在，但关联 Material 已丢失 | 使用 `/audit` 页面进行扫描，记录 Findings 后通过命令行清理。 |
| **Parsed File Missing** | `mineruStatus=completed` 但 MinIO 中找不到 MD 文件 | 可能是物理文件被误删，需重新触发解析。 |

---

## 4. 注意事项

1.  **只读诊断**：前端展示的“健康”或“需审计”仅为基于状态位的逻辑判断，不直接修改底层数据。
2.  **非替代关系**：任务诊断用于**快速发现**个案问题；一致性审计 (`/audit`) 用于**全面排查**系统级数据不一致。
3.  **人工核验优先级**：对于 `review-pending` 状态的任务，即便诊断显示健康，也必须通过人工核验 metadata 准确性后方可发布。
